# OS Checklist


# UI Guidelines
## Requirements
1. - [ ] Give some level of whitespace around the UI.
2. - [ ] Have accessibility options.


# Data Dictionary
## Template
1. - [ ] Table Name
2. - [ ] Field Name
3. - [ ] Field Definition
4. - [ ] Data Type & Length
5. - [ ] Format / Validation Rules
6. - [ ] Primary Key
7. - [ ] Foreign Key
8. - [ ] Sample Data


# Use Case Diagram
## Template
1. - [ ] Actors to represent entities or account types.
2. - [ ] Frame to encapsulate all functions.
3. - [ ] Ovals for functions.
4. - [ ] Arrows to show the direction of the interaction.


# Data Flow Diagram
## Level 0
### Template
1. - [ ] Central oval to represent the system.
2. - [ ] Rectangles to represent entities.

## Level 1
### Template
1. - [ ] Circles to represent processes.
2. - [ ] Rectangles to represent entities.
3. - [ ] Yellow rectangles that are thin to represent datastores. (or use the symbol)
4. - [ ] Arrows to show the direction of the flow, no double-sided arrows allowed.
